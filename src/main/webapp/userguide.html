User guide

PEALT input consists of text statements delimited by \n (NEWLINE) characters. The sections of the input are divided by keywords, POLICIES, POLICY_SETS, CONDITIONS, DOMAIN_SPECIFICS (optional), and ANALYSES. The details of the language grammar is documented in the aforementioned technical report. This guide uses the sample generated by the "Constant-score sample" button to help explain the input syntax.

POLICIES

Multiple input policies can be defined in this section. A valid policy may be, 

b1 = min ((q1 0.2) (q2 0.4) (q3 0.9)) default 1

Here, b1 is the policy name, min is the operator, followed by zero or more rules in a pair of brackets. Each rule is a tuple of (predicate name, score). Finally, the statement ends with the "default" keyword and a default score.

A predicate is a boolean value, a true value means the associated score is applicable, otherwise it means the tuple has no effect in the container policy. If all predicates are false, then the policy is assigned the default score. If a predicate name appears in more than one policies, it simply means the same boolean variable controls application of associated scores in those policies.

If the operator is changed to +, *, or max, the score of the policy is aggregated accordingly.

Note that while voilating this does not cause an error, the policy name should be unique. Internally, policies are held in a map with polcy names as the key, thus the policies will be overridden by later ones of the same name.

At the risk of spelling out the obvious, this statement says b1 is evaluted to the minimum of all scores where the predicates are true. If all predicates are false, then b1 is assigned the default score 1.

PEALT usage constraints

We now describe the illicit combinations, which we want to automatically disable in future version of the tool:

Usage constraint 1: explicit synthesis should not be used when any + policy occurring within an analysis contains some negative or non-constant score or when any such * policy has scores not in the unit interval [0,1]; it should also not be used when any min or max policy occurring in an analysis contains some non-constant score (negative but constant ones are OK then).

Usage constraint 2: symbolic synthesis should not be used when any min or max policy occurring within an analysis contains some non-constant score (negative but constant ones are OK then).

For example, it is safe to use explicit synthesis for constant scores in the unit interval. For example, it is safe to use symbolic synthesis for both positive and negative scores when only + and * policies have non-constant scores.

Additionally, pretty printed output is currently not available for inputs with domain specifics, and symbolic synthesis results. This problem will be rectified in due course.
